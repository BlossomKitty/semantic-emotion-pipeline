# 局限与方法边界

本文说明当前自我复盘系统的主要局限、解释边界和后续改进方向。系统目标是辅助个人回看长期文本中的主题、情绪和证据线索，不是心理诊断、人格判断或自动生成结论的工具。

## 1. 数据来源边界

当前系统主要处理个人微博文本。微博文本本身具有明显的表达场景偏差：

- 它是公开或半公开表达，不等同于完整生活记录。
- 它更容易保留可被发布、可被表达、可被回看的内容。
- 沉默、未发布内容、线下事件、关系背景和真实行动结果通常缺失。
- 不同年份的发布频率、平台使用习惯和表达风格可能发生变化。

因此，系统只能分析“文本中呈现出的轨迹”，不能直接还原完整人生经历或心理状态。

## 2. 样本选择与证据包局限

系统通过 LLM 伪标签、本地 keep 模型和证据包筛选来减少低信号文本，但这并不意味着保留下来的文本就是“全部重要文本”。

当前证据包存在几类局限：

- 首轮伪标签只覆盖部分样本，不是全量人工标注。
- 本地 keep 模型学习的是 LLM 伪标签的保留倾向，因此会继承 LLM 的判断偏差。
- 长文本可能因为信息密度较高而被兜底保留，但长文本不一定天然更重要。
- 短文本、隐喻文本、低频但关键的转折文本，可能被低估。
- 证据包更适合做解释材料索引，不适合作为严格统计样本。

因此，证据包应被理解为“候选证据集合”，而不是客观完整的成长轨迹。

## 3. LLM 先验与主题框架局限

当前报告生成中存在一个重要局限：模型和报告模板预先给出了若干成长主线主题，例如科研与承担、关系支点、城市迁移、社会结构、文学历史、创作技术等。

这种设计有助于报告结构稳定，但也会带来问题：

- 主题是先验设定的，不一定完全来自文档自身的自然分布。
- 如果文本实际呈现出其他主线，预设主题可能遮蔽或压缩这些主线。
- 预设主题会影响后续证据归类，使报告更容易围绕既有框架组织，而不是完全从材料中生长出来。
- 不同主题之间可能存在重叠，例如文学历史观察、社会结构理解和自我叙事并不总能清晰分开。

因此，当前主题框架更适合作为解释脚手架，而不是最终主题发现结果。

## 4. 缺少文档级元数据

当前系统主要依赖文本内容、发布时间、长度、LLM 标签、情感分数和 embedding 表示。它缺少更丰富的文档级元数据，例如：

- 发布场景和平台上下文。
- 是否为转发、评论、原创、长文或片段。
- 与真实事件、学习阶段、城市变化、关系变化的外部对应关系。
- 文本的受众、写作目的和发布后的互动反馈。
- 同一时期非微博材料，如日记、论文、作品、聊天记录、项目记录等。

缺少这些元数据会导致解释层只能根据文本内部线索推断，容易把表达形式误读为生活事实，也容易把阶段性表达误读为稳定状态。

## 5. 主题分析与成长轨迹之间的断裂

当前系统中，主题分析和成长轨迹解释仍然存在逻辑断裂。

可视化中的主题/技能热力图主要来自 TF-IDF + NMF 等无监督文本特征；成长轨迹报告则更多依赖 LLM 对证据包的解释。这两部分之间没有形成严格的同一套因果链或推理链。

具体表现为：

- 主题热力图反映的是文本词项和主题特征，不直接等于成长阶段。
- LLM 报告使用的是证据包和解释 prompt，不一定严格引用主题模型结果。
- 某些主题在图表中明显，但未必会进入报告主线。
- 某些报告主线来自解释框架或证据阅读，不一定能在主题热力图中直接对应。

因此，目前的主题分析更适合作为探索性辅助，而不是成长轨迹解释的严格证明。

## 6. 情感模型边界

当前情感趋势来自中文 RoBERTa 情感模型，主要输出文本的情感极性分数。它的边界包括：

- 情感极性不等于心理健康状态。
- 正向分数高不代表生活状态更好，负向分数高也不代表心理问题。
- 文学评论、讽刺、反语、复杂叙事和历史评价容易被情感模型误判。
- 月度均值或中位数只能显示文本表达倾向，不能直接解释现实事件。

因此，情感图只应作为文本情绪信号，不应作为诊断依据。

## 7. 语义地图边界

语义地图当前使用 BGE embedding + PCA。它适合展示文本在高维语义空间中的粗略分布，但可解释性有限。

主要局限包括：

- PCA-1 和 PCA-2 只是最大方差方向，不天然对应“情绪轴”“关系轴”或“创作轴”。
- 高维 embedding 被压缩到二维后会损失大量信息。
- 点与点之间的距离只能粗略参考，不能被过度解释。
- 语义地图更像探索入口，不是严格的分类、聚类或因果解释。

因此，语义地图不应单独承担结论功能。真正的解释仍应回到证据包、原文、标签和人工复核。

## 8. 可视化统计口径边界

不同图表使用的统计口径并不完全相同：

- 事件密度图更接近全量清洗文本的时间分布。
- 情感趋势图基于全量或缓存情感分数。
- keep 趋势图基于已标注样本和模型预测。
- 心路动作与证据类型图应基于最终保留证据，而不是全部伪标签样本。
- 主题/技能热力图更偏向无监督文本探索。

因此，不同图表之间不能简单相互证明。它们分别回答不同问题：文本何时密集、情绪表达如何变化、哪些证据被保留、保留证据呈现哪些动作和类型、文本主题如何分布。

## 9. 解释输出边界

LLM 解释报告应遵守以下边界：

- 不做临床诊断。
- 不根据单条文本判断人格。
- 不把诗性表达直接等同于心理事实。
- 不把关系对象神化、病理化或工具化。
- 所有强解释必须绑定证据，并给出反解释。
- 证据不足时应明确说证据不足。
- 报告中的阶段划分是解释性组织，不是客观历史分期。

报告应被看作“可讨论的复盘草稿”，而不是最终判断。

## 10. 后续改进方向

后续可以从以下方向减少上述局限：

- 增加文档级元数据，例如发布场景、文本类型、外部事件和阶段标记。
- 引入更多材料来源，例如项目记录、作品记录、日记或阶段总结。
- 将主题发现从预设主线改为“材料驱动主题 + 人工命名 + LLM 解释”的组合流程。
- 让主题模型输出和成长报告生成之间建立显式引用关系。
- 对语义地图增加 explained variance ratio、聚类摘要和代表文本抽样。
- 对证据包增加人工复核入口，允许用户修正 keep、evidence_type 和 inner_action。
- 区分展示图、探索图和解释图，避免把探索性可视化当成结论。

当前系统的合理使用方式，是把它作为个人长期文本复盘的证据整理和解释辅助工具。它能帮助发现问题、组织材料和生成讨论线索，但最终解释仍需要用户结合现实经历、上下文和人工判断来完成。
